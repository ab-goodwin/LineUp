I want to migrate my authentication system from Passport Local + bcrypt to Supabase Authentication.

This is a production application. Preserve all existing functionality. Make the smallest possible changes. Do NOT redesign the application.

---

## DO NOT

* Do NOT redesign the project structure.
* Do NOT delete database tables.
* Do NOT rename routes.
* Do NOT modify any dance/session/statistics/business logic.
* Do NOT remove existing user profile information.
* Do NOT convert every table to use Supabase UUIDs.
* Do NOT make assumptions if something is unclear. Leave comments instead.

# Authentication Architecture

Supabase Auth will ONLY manage authentication.

My existing users table will continue to store all application profile information.

Passwords must no longer be stored in my users table.

Authentication should work like this:

Supabase Auth

* Email
* Password
* Auth UUID

↓

users table

* Existing numeric/UUID id (leave unchanged)
* Username
* Email
* First Name
* Last Name
* Avatar
* Homepage Stats
* Location
* Phone
* etc.

The users table remains the source of truth for application profile data.

Supabase Auth only stores authentication.

# Modify the users table

Add ONE new column:

supabase_auth_id UUID NULL

Do NOT remove any existing columns.

Leave passwordHash in place temporarily until the migration is complete.

Do NOT delete any existing user data.

# Registration

The registration page should continue asking for:

* Username
* Email
* Password
* First Name
* Last Name

When a new account is created:

1. Create the user in Supabase Auth using:

supabase.auth.signUp()

2. Save the returned Auth UUID into:

users.supabase_auth_id

3. Save the rest of the profile information into my existing users table.

4. Do NOT store the password in my database.

5. Remove any bcrypt hashing during registration.

---

# Login

IMPORTANT:

I want users to continue logging in with:

Username
Password

NOT email.

The login flow should work like this:

1. User enters username.

2. Query my users table to retrieve the user's email.

3. Call:

supabase.auth.signInWithPassword()

using that email and the entered password.

4. If authentication succeeds:

Retrieve the matching row from the users table.

5. Continue using my existing internal users.id throughout the rest of the application.

Do NOT convert the rest of the application to use Supabase UUIDs.

Replace Passport session handling with Supabase Auth session handling.

* supabase.auth.getSession()

# Current User

Wherever the application currently retrieves the logged-in user, update the logic to:

1. Get the authenticated Supabase user.

users.supabase_auth_id

# Logout

supabase.auth.signOut()

Email Address
